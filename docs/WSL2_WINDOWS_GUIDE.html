PHASE 0 Verify Your WSL2 Environment ⏱ 10 min
Confirm WSL2 is running correctly
You need WSL2 (not WSL1) — Docker Desktop only works with WSL2

Open PowerShell as Administrator (right-click Start → Windows Terminal as Admin) and run:

Don't have Ubuntu installed yet? Run wsl --install in PowerShell as Admin, then restart your PC. After restart, Ubuntu will finish installing and ask you to create a username and password.

Understand the golden rule: ALWAYS work in WSL2 filesystem
The #1 mistake Windows developers make with Docker
❌ DON'T — Windows filesystem
  • Storing project in C:\Users\you\...
  • Docker volume mounts break silently
  • File watching (hot reload) doesn't work
  • 10-100x slower than native Linux I/O
  • Line endings (CRLF) corrupt shell scripts
✅ DO — WSL2 filesystem
  • Storing project in ~/projects/... (inside Ubuntu)
  • Docker volumes mount perfectly
  • File watching works correctly
  • Native Linux I/O speed
  • Unix line endings (LF) always correct
Ubuntu (WSL2) — set up your project home

PHASE 2 Set Up Python and Run Tests ⏱ 15 min
Install Python in WSL2 and set up a virtual environment
WSL2's Python is separate from Windows Python — always use a venv
Ubuntu terminal

Python 3.12.x

PHASE 3 Build and Run with Docker ⏱ 20 min
Build the Docker image from WSL2
Docker Desktop on Windows runs containers through WSL2 — this is why your project must be in the WSL2 filesystem
Ubuntu terminal — from ~/projects/devops-demo-api

$ docker build -t devops-demo-api:local .

 => [runtime 1/3] COPY --from=builder /install /usr/local

# Verify it runs as non-root (security check)

⚠️
Slow first build is normal. Docker pulls the base Python image (~150MB) on first run. Subsequent builds are much faster because Docker caches layers. The whole point of ordering COPY requirements.txt before COPY main.py in the Dockerfile is to cache the pip install layer.

Add GitHub Secrets for Docker Hub
The pipeline needs credentials to push your Docker image

GitHub → Your repo → Settings → Secrets and variables → Actions → New repository secret

First, get a Docker Hub access token:

  1. Go to hub.docker.com → log in → click your profile icon
  2. Account Settings → Security → New Access Token
  3. Name it github-actions, permission: Read & Write
  4. Copy the token — you only see it once

Then add these two secrets to GitHub (minimum needed for the build job):

GitHub Secrets to add

TROUBLESHOOT Common WSL2 Issues & Fixes reference
Error / Symptom Fix
docker: permission denied Run sudo usermod -aG docker $USER then close and reopen the terminal completely (not just a new tab)
docker: Cannot connect to Docker daemon Docker Desktop isn't running. Open Docker Desktop on Windows and wait for it to fully start (whale icon stops animating)
Port 80 already in use Something on Windows is using port 80 (often IIS). Either stop it (net stop w3svc in PowerShell) or change the port in docker-compose.yml to "8080:80" and use http://localhost:8080
/bin/bash^M: bad interpreter Windows line endings in a shell script. Run: dos2unix server-setup.sh
ModuleNotFoundError: flask Virtual environment isn't active. Run source .venv/bin/activate first
pytest: command not found Install in the venv: pip install pytest pytest-cov
git push asks for password, then fails GitHub no longer accepts passwords. Use gh auth login or create a Personal Access Token and use it as the password
code . doesn't open VS Code Install VS Code on Windows (not inside WSL2), then run code --install-extension ms-vscode-remote.remote-wsl in PowerShell
Docker build extremely slow Make sure your project is in ~/ (Linux filesystem), NOT in /mnt/c/.... Cross-filesystem I/O is 10-100x slower
nginx: upstream timed out The Flask container isn't ready yet. Wait for the (healthy) status in docker compose ps before testing
